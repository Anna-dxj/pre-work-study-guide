HTML

  • HTML - language that provides basic content structure to all webpages
  • Elements - types of content on a page
    • Composed of opening and closign tags indicated by <>
  • Head elements - contains information (metadata) that helps browser to render a page correctly
    • Elements are not visible to user
    • Metadata - data about a webpage that are typically not seen by user but contains important information about accessibility, search engines, performances
  • Body elements - contains content shown to the user


CSS

The Basics

  • CSS - language that gives content style
  • Defines things like colors, aesthetics, visual layouts, organization, etc.
  • Inline CSS - Uses style attrribute with CSS to a particular HTML element only
    • Applies to specific HTML tags
    • Good for quick & permanent changes
    • Less flexible than others; each inline must be edited individually whenever make design change
  • Internal style sheet - style sheet embedded directly into HTML (use style tag in head element)
    • Rules only apply on that page
    • Single changes to CSS rule will apply to all tagged page elements
  • External style sheet - completely different file and must be linked to HTML code in head using link tag
    • Most common and preferred method
    • Allows developers to keep all CSS rules in separate file and makes changes easier


CSS Rules

  • Selector - defines the element or attributes to which rules/declarations will apply
  • Declarations - Composed of CSS proprty and its value
  • CSS Box Model - concept that every HTML element is a box, which consist of margins (outside elemtnt) and element's boarders, padding, and content
    • margin - indicates how much space want outside elemtn; not visible to page, just buffer
    • padding - adds space around content inside an element; increasing padding makes it bigger
  • , combines two sectors into single CSS rule, such that declaration will affect both selectors
  • * indicates rule will be applied to all elements visible on page


Git

Commandline

  • ls: list of files and directories
  • cd: change working directory
  • pwd: print working directory
  • cd ..: go back one directory
  • mkdir: creates new directory
  • touch: creates new file
  • vs. git commands: git commands always start with "git", but are also put into command line


GitHub Workflow

  • Main/master branch - default work branch; shouldn't do any coding directly into main branch to keep track of changes made to it
  • Feature branches - branches created to work on code directly or add anything to project; later merged with main branch
  • Pul request (PR) - request used to merge a braench with main branch
  • Workflow:
    1. Create branch with short descriptive name (git checkout -b "branch-name")
    2. Make desired changes to repository
    3. Add modifications to staging area (git add -A)
    4. Commits (git commit -m "commit")
    5. Sync local and remote branches (git pull origin main)
    6. Push changes made to local branch to remote branch (git push origin "branch-name")
    7. Create pull request and address review comments
    8. Merge pull request and delete branch


JavaScript

Basics

  • JavaScript - high-level dynamic programming language that handels logic and allows users to interact with a website
  • Reads top to bottom
  • Can be used to power effects, interactivity, data communications, etc.; makes webpage interactive

Variables

  • Variable - named container that allows storage of data in code; assigned wtih =
  • Array - single variable that is used to hold a group of data
    • Typically used to hold data that is related in some way
    • Can hold different types of data (strings, numbers, etc.)
  • Index - the number associated with the location of a piece of data within array
    • Can be used to access specific piece of information in array
    • Zero-indexed (ie. starts at 0), sequential


Conditionals

  • One of teh ways to control the order in which code is executed if at all
  • If statement introduces decision-making into code
  • Code will only be executed if condition is met

Loops & for loops

  • Loops - way to write code that repeats without actually repeating it
  • For loops1 - use predictable pattern of indices to perform a task on all items in array by allowing single code block to be executed repeatedly
  • Use another statement (ex. conditional) to interrupt control flow by executing block of code specific number of times
  • Can tell computer to go through each index in array and log them to console until none are left
    • Need index and array length (needs to know hoiw many times code will run until there are no items left in array)
  • Parts of a loop: starting point of loop; condition; statement that allows the array to iterate over each item


Functions

  • Function - a set of instructions that tells computer how to perform a certain task
  • Functions don't automatically execute when JavaScript file is run
  • Must call fu nction by name to execute it; makes it easily reusable
    • Can call it as many times as needed
    • Can control where in control flow task is executed if at all
  • Function followed by (), where define perameters
