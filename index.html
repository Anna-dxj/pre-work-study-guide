HTML

    The Basics

  • Head: element contains information about the webpage
  • Body: element represents the visible contrent shown to the user
  • Start commands with open <, end command with />


    • HTML Tags

  • head/body: designate as head/body elements
  • !--: Comment
  • !DOCTYPE: defines document type
  • title: title of webpage
  • style: defines CSS styles associated
  • base: defines URL for page
  • links: connects external resource for HTML doc
  • meta: defines metadata (ex: character set, description, keywords, author, viewport, etc.)
  • script: embeds client-site scripts in HTML doc
  • ul/ol/li: unordered list/orderd list/ list items
    • Note to embed a list item, add ul/ol tag inside previous ul/ol tag
  • h1/2/3/4/5/6: level of headding designation
  • p: paragraph
  • img: image
  • be: line break
  • a: "anchor"; creates link for same/different webpage
  • Text formatting:
    • b: bold text
    • i: italicize
    • del: deleted text
    • sub: subscript
    • sup: superscript


CSS

    The Basics

  • Margin indicates how much space want outisde of an element.
  • A padding indicates how much space we want around the content inside an element
  • Inline CSS: adds style elements directly in HTML file
    • Pros: Quick and permanent changes, apply to specifc HTML tags
    • Cons: Less flexible than external and intaernal styles; must be edited individually when make design change
  • Internal style sheet: rules that only apply to that page
    • Note: can configure CSS class and IDs to style multiple elements in page code
  • External style sheet: most common way to style a webpage that allows developers to keep CSS rules in a different file
    • Pros: Easy to make design changes
    • Notes: Use link in HTML head element


    CSS Rules

  • div {}: selector
    • Selector: defines element/attributes to which rules/declarations will apply
    • Declarations: CSS property want to apply and value of property
  • margin: property that indicates how much space want outside of a given element; not visible on a page
  • padding: property that adds space around teh content jinside and element; makes an entire element biger ultimately
  • *: "special wild-card symbol" to indicate to apply rule to all elements visible on page
  • Combine two selectors into one CSS rule with ,
  • text-align: horizontal alignment of text in element
  • img: image properties (display, height, width, margins, etc.)
  • ul: affecting UL elements (padding, font-size)
  • p: affecting p elements (text-alignment, font-size)
  • .[class]: indicates what's following the period as a class sector (any HTML element assigned to that class will gain property assignments)


    • Adding CSS to HTML

  • Code: link rel="stylesheet" herf="./[folder]/[style sheet name]
  • rel: specifies the relationship between current document and linked document resource
  • herf: specializes location (URL) of external resource
  • In the prework study guide example, link it after the last meta before title elements in head element


Git

    Using the Command Line

  • ls: list of files and directories
  • cd "name": change directory to "name"
  • cd ..: navigate backwards one directory at a time
  • pwd: print working directory
  • mkdir "name": create new directory
  • code .: open folder in VS code


    • Git commands

  • git status: checks what branch currently on
  • git checkout -b "branch-name": creates new branch and switches to it
  • git add -A: Adds all (-A) modification in working branch area
  • git commit -m "commit": Adds description of changes made to branch
  • git pull origin main: check if local branch is in sync with base branch on GitHub
  • git push origin "branch name": push changes of "branch name" to remote Github branch


    • GitHub Workflow

  • Ensure GitHub repository is in sync with local drive
  • Default is main/master; work off of feature branches
  • Feature branch: make commits of changes to code
  • Push up changes to feautre branch in GitHubh by opening PR (pull request)
  • Wait for review and apprival before merging with main branch


    • Git Version Control

    1. git status
    2. git add -A
    3. git commit -m "commit"
    4. git pull origin main
    5. git push origin "branch name"
    6. Compare and pull request on GitHub


JavaScript

    Terms

  • Variable (var): named container that allows developer to store data in code
  • Control flow: order in which a computer executes code in a script
  • Array []: single variable that is used to hold a group of data and is typically used to hold data that is somehow related
  • Index: Numbers associated with location of data in array; first item always 0


    • Basics

  • Variable names must be unique
  • = assigns specific data to variable
  • Strings constrained by "" and are used to represnt and manipulate a sequence of characters
  • Link JavaScript in the Body element
  • Code runs from first statement at top of page down to last statement at bottom of page


    • Conditionals

  • Truthy: when a condition is true
  • If condition is true, will do action asked of it
  • If condition isn't true, computer ignores
  • else if: alternate condition


    • For Loops

  • Uses a predictable pattern of indicies to perform a task on all items in an array by allowing a single code block to be executed repeatedly
  • Uses another statement (ex: conditional) to interrupt control flow by executing a block of code for a specific number of times
  • Elements
      1. Starting point for loop (ex: var x = 0)
      2. Condition (ex: x < shapes.length)
      3. Array integrate over each item (ex: x++)
